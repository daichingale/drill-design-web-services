# ベータ版機能一覧

このドキュメントは、現在ベータ版として提供されている機能と、今後の改善予定をまとめています。

## 🎵 音楽分析（AI）機能

### 現在の実装状況
- ✅ 簡易版（冒頭30秒のみ、高速）
- ✅ 高精度版（全曲解析、テンポ変化検出）
- ✅ BPM検出（複数のフォールバック方法を実装）
- ✅ セクション検出（イントロ、Aメロ、Bメロ、サビなど）
- ✅ テンポ変化検出（高精度版のみ）

### 既知の制限事項
1. **BPM検出の精度**
   - 一部の音楽ファイルでBPMが0になることがある（フォールバックで120 BPMを使用）
   - 複雑なリズムパターンやテンポ変化が多い楽曲では精度が低下する可能性

2. **セクション検出**
   - 現在は簡易的なセクション検出のみ
   - セクションのラベル（イントロ、Aメロなど）は自動判定されない（手動で設定が必要）

3. **対応ファイル形式**
   - MP3, WAV, M4A, FLAC などに対応
   - ただし、一部のコーデックでデコードエラーが発生する可能性

4. **パフォーマンス**
   - 高精度版は長い楽曲（5分以上）で30秒以上かかることがある
   - Python 3.14では`librosa`が使用できない（Python 3.12以下が必要）

### 今後の改善予定
- [ ] BPM検出の精度向上（より多くのフォールバック方法）
- [ ] セクションの自動ラベル付け（イントロ、Aメロ、Bメロ、サビの自動判定）
- [ ] 拍子の自動検出（4/4以外の拍子にも対応）
- [ ] より高速な解析アルゴリズムの実装
- [ ] Python 3.14対応（`librosa`の代替ライブラリの検討）

## 🤖 学習・提案機能

### 現在の実装状況
- ✅ ドリルデータの保存機能
- ✅ 学習統計の表示（保存済みドリル数、よく使う隊形など）
- ✅ セクション別提案ボタン（UIのみ、実装は未完了）

### 既知の制限事項
1. **学習アルゴリズム**
   - 現在は基本的な統計計算のみ（パターン抽出は未実装）
   - 機械学習モデルは未統合

2. **提案機能**
   - セクション別提案ボタンはUIのみで、実際の提案機能は未実装
   - パターンマッチング機能は未実装

3. **データ永続化**
   - 現在はメモリ上に保存（サーバー再起動で消える）
   - データベースへの保存は未実装

### 今後の改善予定
- [ ] パターン抽出アルゴリズムの実装
- [ ] セクション別提案機能の実装
- [ ] パターンマッチング機能（「似た動きを提案」）
- [ ] データベースへの永続化
- [ ] 機械学習モデルの統合（オプション）

## 📝 注意事項

### ベータ版機能の使用について
- これらの機能は**実験的な段階**にあります
- 予期しない動作やエラーが発生する可能性があります
- 重要なデータは必ずバックアップを取ってください

### フィードバック
- バグ報告や改善提案は大歓迎です
- 問題が発生した場合は、エラーメッセージと再現手順を共有してください
